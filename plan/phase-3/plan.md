# Phase 3 — Snooze / 重复 / 下一个 + 日志 + 通知

> **Related overview sections**: [Section 4 (状态机)](../overview.md#状态机), [Section 7.4 (超时策略)](../overview.md#74-超时策略), [Section 9 (日志)](../overview.md#日志)

## 目标

完善队列控制 + 可观测性

## Tasks

1. 实现 snooze 逻辑（"稍后" → 放回队尾 + 延迟 120s）
2. 实现"下一个"/"重复"/"取消"/"状态"命令
3. 实现完整日志系统（JSONL + 轮转：每天一个文件，保留 30 天）
4. macOS 通知集成（`hs.notify`）：事件过期时弹通知
5. 二次确认 UI 完善（撤销窗口 UI 和可配置性）
6. 配置热重载：修改 config.lua 后自动生效

## 验收标准

- "稍后" → 2 分钟后重新播报
- "下一个" → 播下一事件
- "重复" → 重新播报当前事件
- "取消" → 标记 expired
- "状态" → 播报队列概况
- 日志文件按天轮转，记录完整
- 事件过期时弹出 macOS 通知

## 预计工作量

约 6-8 小时
