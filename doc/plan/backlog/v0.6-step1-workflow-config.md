# v0.6 Step 1: Workflow Config + Agent Dispatch

## Summary

让 Flywheel 从 generic 调度器进化为 **project-aware 调度器**。每个被管理的项目 repo 在 `.flywheel/config.yaml` 中定义 agent dispatch 规则，Flywheel 在 dispatch 时读取配置，自动选择正确的 agent 和 workflow。

## Background

### 现状问题

`Blueprint.run()` 对所有 issue 构建相同的 generic prompt（`packages/edge-worker/src/Blueprint.ts:186-196`）：
```typescript
const prompt = `Implement ${hydrated.issueId}: ${hydrated.issueTitle}.\n\n${hydrated.issueDescription}`;
const systemPrompt = "You are working on a Linear issue. Follow these steps: ..."  // 7 generic steps
```

`SkillInjector`（`packages/edge-worker/src/SkillInjector.ts`）只注入 5 个 hardcoded Flywheel skill templates，不知道项目有自己的 agents/domains/skills。

以 GeoForge3D 为例，项目有 7 个 agent、5 个 domain、18 个 skill，但 Flywheel 对此一无所知。

**此外，`setupComponents()` 目前不调用 `ConfigLoader`**——`.flywheel/config.yaml` 加载尚未接入运行路径。`skillsConfig` 在 setup 中被传为 `undefined`。本 Step 需要先完成 config 加载接入。

### 目标

1. 接入 `ConfigLoader`，在 `setupComponents()` 中加载 `.flywheel/config.yaml`
2. 扩展 `FlywheelConfig`（`packages/config`），新增 `agents` section
3. `Blueprint.runInner()` 内部匹配 issue → agent（hydrate 之后、prompt 构建之前）
4. 注入对应 agent 的 executor 定义和 domain 配置到 prompt

### 参考

- OpenAI Symphony 的 `WORKFLOW.md`（YAML front matter + Markdown prompt）
- GeoForge3D 已有的 `.claude/agents/`, `.claude/domains/`, `.claude/skills/` 体系

## Design

### 扩展 FlywheelConfig（`packages/config/src/types.ts`）

在现有 `FlywheelConfig` interface 中新增 `agents` section。**Step 1 只保留有即时消费者的字段**：

```typescript
/** Agent dispatch configuration — Step 1 minimal */
export interface AgentConfig {
  /** Relative path to agent executor file (e.g., .claude/agents/backend-executor.md). REQUIRED. */
  agent_file: string;
  /** Relative path to domain config file (e.g., .claude/domains/backend.md). Optional. */
  domain_file?: string;
  /** Dispatch matching rules */
  match: {
    /** Linear labels that map to this agent (case-insensitive) */
    labels: string[];
    /** Keywords fed to Haiku classifier as hints when no label match */
    keywords: string[];
  };
}

// Future fields (NOT in Step 1 — no consumers yet):
// flow: 'engineer' | 'design' | 'product' | 'qa';
// land_command?: string;

/** Extended FlywheelConfig — add to existing interface */
export interface FlywheelConfig {
  // ... existing fields ...
  /** Agent dispatch rules (project-aware). Optional for backward compat. */
  agents?: Record<string, AgentConfig>;
  /** Default agent name when no match. Falls back to generic prompt if undefined. */
  default_agent?: string;
}
```

**必须同时更新** `packages/config/src/index.ts` 导出 `AgentConfig`。

### Dispatch 算法

**Dispatch 统一在 `Blueprint.runInner()` 内部**（Option B），在 `hydrate()` 之后、prompt 构建之前。不在脚本层做，不给 `BlueprintContext` 加 `agentName` 字段。

```
Blueprint.runInner() flow:
  1. Worktree setup (existing, line 125-149)
  2. Git preflight (existing, line 152-153)
  3. PreHydrator.hydrate() (existing, line 156) → 获得 labels, title, description
  4. [NEW] AgentDispatcher.dispatch(hydrated) → DispatchResult | null
  5. Skill injection (existing, line 163-183)
  6. [MODIFIED] Prompt assembly:
     - if DispatchResult → 读取 agent_file/domain_file → 结构化 prompt
     - if null → 现有 generic prompt（不变）
  7. Runner execution (existing)
```

`AgentDispatcher` 通过 Blueprint constructor 注入（可选参数，向后兼容）。

### Label 匹配 + Haiku 分类

```
1. 如果 config.agents 为空或 AgentDispatcher 为 null → 返回 null（generic 流程）
2. Label 精确匹配：遍历 agents，检查 hydrated.labels ∩ agent.match.labels（case-insensitive）
3. 如果无匹配 → Haiku 快速分类：
   - 复用 AnthropicLLMClient（packages/claude-runner/src/AnthropicLLMClient.ts）
   - 参考 HaikuTriageAgent（packages/edge-worker/src/decision/HaikuTriageAgent.ts）的 fenced context 模式
   - Prompt（含 UNTRUSTED DATA fence）:
     ```
     Classify this Linear issue into exactly one of these agent names: [backend, frontend, designer, product, qa, devops].

     Each agent handles these types of work:
     - backend: [keywords from config]
     - frontend: [keywords from config]
     ...

     <issue_context>
     Title: {title}
     Description: {first 500 chars of description}
     </issue_context>

     Reply with ONLY the agent name, nothing else.
     ```
   - 输出验证：只接受 allowlist 中的 agent name，其他一律 fallback
   - 无 ANTHROPIC_API_KEY 时直接 fallback（和现有 Decision Layer 降级路径一致）
4. Fallback → config.default_agent
5. 完全无匹配 → 返回 null → generic prompt
```

### Prompt 注入策略

**增强（additive），不替换**。agent/domain 内容插入到现有 generic system prompt **之前**，保留所有基线行为约束：

```typescript
// Blueprint.runInner() — prompt assembly
const dispatchResult = this.agentDispatcher?.dispatch(hydrated) ?? null;

// 基线 prompt（始终使用）
const prompt = `Implement ${hydrated.issueIdentifier}: ${hydrated.issueTitle}.\n\n${hydrated.issueDescription}`;

// 基线 system prompt（始终保留）
const baseSystemPrompt = [
  "You are working on a Linear issue. Follow these steps:",
  "1. Read the codebase and understand the context (CLAUDE.md, relevant files).",
  "2. Implement the requested changes following TDD.",
  "3. Create a feature branch, commit your changes.",
  "4. Push the branch and create a GitHub PR.",
  "5. Verify CI passes. If CI fails, fix and push again.",
  "6. When all work is complete, stop and wait.",
  "Do not ask questions — implement your best judgment.",
].join('\n');

// Agent context（additive — prepend before base system prompt）
let agentContext = '';

if (dispatchResult) {
  const { agentConfig } = dispatchResult;
  const agentContent = await readAgentFile(cwd, agentConfig.agent_file);

  if (agentContent) {
    // agent_file 读取成功 → 构建 agent context
    const domainContent = agentConfig.domain_file
      ? await readAgentFile(cwd, agentConfig.domain_file)
      : null;

    agentContext = [
      '## Agent Role',
      agentContent.slice(0, 40_000),
      '',
      domainContent ? `## Domain Config\n${domainContent.slice(0, 10_000)}` : '',
      '',
    ].filter(Boolean).join('\n');
  } else {
    // agent_file 缺失/为空 → warning, agentContext 留空（使用纯 baseSystemPrompt）
    console.warn(`[Blueprint] Agent file not found: ${agentConfig.agent_file}, using generic prompt`);
  }
}

// 最终 system prompt = agent context (if any) + base system prompt (always)
const systemPrompt = agentContext
  ? `${agentContext}\n## Baseline Rules\n${baseSystemPrompt}`
  : baseSystemPrompt;
```

**关键保证**：
- 无论是否有 agent dispatch，基线 system prompt **始终保留**（CLAUDE.md 读取、TDD、stop and wait）
- agent/domain 内容是 **prepend**，不是 replace
- agent_file 缺失 → 回退到纯基线 prompt（和当前行为完全一致）
- domain_file 缺失 → 仅缺少 domain context，不影响执行

### 路径安全

路径验证和文件读取 **分离**，避免 `realpath()` 在文件不存在时抛 ENOENT 导致 fallback 失效：

```typescript
/**
 * 验证相对路径的安全性（不做 realpath — 文件可能不存在）。
 * 在 ConfigLoader.validate() 中调用。
 */
function validateAgentPath(relativePath: string): void {
  if (path.isAbsolute(relativePath)) {
    throw new Error(`Agent path must be relative: ${relativePath}`);
  }
  if (relativePath.startsWith('..')) {
    throw new Error(`Agent path must not escape repo: ${relativePath}`);
  }
}

/**
 * 安全读取 agent/domain 文件。
 * 路径验证 → resolve → containment check → read。
 * 文件不存在时返回 null（不抛异常）。
 */
async function readAgentFile(repoRoot: string, relativePath: string): Promise<string | null> {
  validateAgentPath(relativePath);

  const resolved = path.resolve(repoRoot, relativePath);

  // Containment check（基于 resolve，不依赖 realpath）
  const normalizedRoot = path.resolve(repoRoot) + path.sep;
  if (!resolved.startsWith(normalizedRoot) && resolved !== path.resolve(repoRoot)) {
    console.warn(`[Blueprint] Agent path escapes repo: ${relativePath}`);
    return null;
  }

  try {
    const content = await fs.promises.readFile(resolved, 'utf-8');

    // 如果文件存在，用 realpath 做 symlink containment 二次验证
    const realResolved = await fs.promises.realpath(resolved);
    const realRoot = await fs.promises.realpath(repoRoot);
    if (!realResolved.startsWith(realRoot + path.sep)) {
      console.warn(`[Blueprint] Agent file symlinks outside repo: ${relativePath}`);
      return null;
    }

    return content || null;  // 空文件视为不存在
  } catch (err) {
    if ((err as NodeJS.ErrnoException).code === 'ENOENT') {
      return null;  // 文件不存在 → graceful fallback
    }
    throw err;  // 其他错误（权限等）→ 上抛
  }
}
```

**Size guard**: 在 prompt assembly 中对 `agentContent` 和 `domainContent` 分别截断（40KB / 10KB）。

### Task 2: AgentDispatcher

**文件**: `packages/edge-worker/src/AgentDispatcher.ts`（新建）

```typescript
import type { AgentConfig } from "flywheel-config";
import type { HydratedContext } from "./PreHydrator.js";

export interface DispatchResult {
  agentName: string;
  agentConfig: AgentConfig;
  matchMethod: 'label' | 'haiku' | 'fallback';
}

export type ClassifyFn = (
  title: string,
  description: string,
  agentNames: string[],
  agentKeywords: Record<string, string[]>,
) => Promise<string | null>;

export class AgentDispatcher {
  constructor(
    private agents: Record<string, AgentConfig>,
    private defaultAgent: string | undefined,  // undefined = no default, return null
    private classifyFn?: ClassifyFn,
  ) {}

  async dispatch(hydrated: HydratedContext): Promise<DispatchResult | null> {
    if (Object.keys(this.agents).length === 0) return null;
    // 1. Label match (case-insensitive)
    // 2. Haiku classify with fenced context (if classifyFn provided)
    //    - Output validated against agent name allowlist
    // 3. If defaultAgent defined and exists in agents → fallback
    // 4. Return null → generic prompt
  }
}
```

**`default_agent` contract**: 在 `FlywheelConfig` 中 `default_agent` 是可选的。`AgentDispatcher` 接受 `string | undefined`。当 `undefined` 时，如果 label 和 Haiku 都没命中，返回 `null`（走 generic prompt）。`ConfigLoader.validate()` 仅在 `default_agent` 有值时验证它存在于 `agents` 中。

- `classifyFn` 构建：在 `scripts/lib/setup.ts` 中使用 `AnthropicLLMClient`
- 遵循 `HaikuTriageAgent` 的 fenced context + UNTRUSTED DATA 模式
- 无 API key → `classifyFn` 为 undefined → 直接 fallback

## Risks

1. **Config 加载失败**: 如果 `.flywheel/config.yaml` 格式错误，应 throw 并终止。如果文件不存在，应静默 fallback。
2. **Haiku 分类准确度**: 用 fenced context + output allowlist 降低误判。不准时 fallback 到 default agent。
3. **Prompt size**: 加了 40KB + 10KB hard limit。正常文件 5-10KB 以内。
4. **向后兼容**: `agents` section 完全可选。没有它的项目走现有 generic 流程，零影响。
5. **路径安全**: `realpath` + containment check + `ConfigLoader` 拒绝绝对路径和 `..` 前缀。
