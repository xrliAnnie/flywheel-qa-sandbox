# Plan: Bridge Discord Identity — 谁发消息给 Annie？

**Version**: v1.21.0
**Issue**: FLY-47 (Channel Contract scope extension)
**Date**: 2026-04-05
**Source**: QA P0 bug — Lead Discord plugin 过滤 Bridge 用 Lead token 发的消息
**Status**: confirmed (Annie 决策 2026-04-05)

## 问题

Bridge 需要向 Lead 的 Discord channels 发送消息，但用 Lead 自己的 bot token 发送会被 Lead 的 Discord plugin 过滤（自己发给自己 = 忽略）。

## 确认方案

Annie 决策：**Chat 走 Lead，Forum 留 Bridge，Control 用 Bridge bot。**

### 三条路径的处理

| 路径 | Channel | 受众 | 方案 | Token |
|------|---------|------|------|-------|
| Control plane | `controlChannel` | Lead agent（机器） | Bridge 直发 | `CLAUDEBOT_TOKEN`（必须与 Lead 不同） |
| Chat relay | `chatChannel` | Annie（人类） | Lead 自己发 | Lead 自己的 bot token |
| Forum | `forumChannel` | Annie（按需看） | Bridge 直发 | Lead 的 bot token（模板化 update） |

### Chat 走 Lead（B 通信模式）

```
Gate question / Stale notification
  → Bridge 写 lead_event + 投递到 Lead controlChannel
  → Lead AI 读到事件
  → Lead 用自然语言在 chatChannel 跟 Annie 对话
```

- Lead 通过 controlChannel 收到所有事件（已实现）
- Lead bootstrap 包含 pending gate questions（已实现）
- Lead 被指示 relay gate questions to Annie（bootstrap 里已有 Action 指令）
- Bridge 不再直接 POST 到 chatChannel

### Forum 留 Bridge（模板化）

- Forum 是 dashboard，显示状态用的，不需要自然语言
- `ForumPostCreator` + `ForumTagUpdater` 继续在 Bridge 运行
- Lead 通过 controlChannel 收到事件通知，知道每个阶段在发生什么
- Forum 内容风格不需要改（Annie 确认模板化是对的）

## 实施变更

1. ~~GatePoller.postToChatChannel()~~ — 移除直接 Discord POST
2. Stale session notification — 改为 lead_event + control channel delivery
3. `bridgeBotToken` 只用于 controlChannel delivery
4. GatePoller dedup bug — 随 postToChatChannel 移除自然消失
5. EventFilter Chat-track gap — `session_started(retry)` 和 `session_completed(approved)` 从 `forum_only` 改为 `notify_agent`，确保 Lead 能向 Annie Chat 通知所有 spec §2.3 定义的触发点

## Codex 审查记录

- **Design doc**: 2 rounds → APPROVED
- **Forum 用 Lead token**: 1 round → CHANGES REQUESTED（access.json 自过滤）
- **Forum 走 B**: 1 round → FORUM_VIA_BRIDGE（产品效果优先）
