# Plan: Approve/Ship 状态机拆分

**Version**: v1.21.0
**Issue**: FLY-58
**Date**: 2026-04-05
**Source**: `doc/plan/new/v2.1-FLY-52-product-experience-implementation.md` §4.1
**Status**: draft

## Summary

拆分 `approve` action：当前 approve 直接触发 `gh pr merge`（ApproveHandler.execute），改为只写 `approved_to_ship` 状态，不执行 merge。merge 留给 Runner /spin ship 阶段。

同时修复 EventFilter：`approved` completion 从 `forum_only` 改为 `notify_agent`（FLY-61 quick fix）。

## Architecture

```
BEFORE: approve → ApproveHandler.execute(gh pr merge) → status "approved" (terminal) → postMergeCleanup
AFTER:  approve → status "approved_to_ship" (intermediate) → [Runner ship checks API] → "completed"
```

## Changes

### 1. `packages/core/src/workflow-fsm.ts`

**WORKFLOW_TRANSITIONS**:
- `awaiting_review`: replace `"approved"` with `"approved_to_ship"` in allowed targets
- Add `approved_to_ship: ["completed", "shelved"]` (intermediate, not terminal)
- Keep `running → approved` for auto_approve path (PR already merged by Runner)
- Keep `approved: []` as terminal (auto_approve still uses it)

**ACTION_DEFINITIONS**:
- `approve.targetState`: `"approved"` → `"approved_to_ship"`

### 2. `packages/teamlead/src/StateStore.ts`

- Add `"approved_to_ship"` to `OUTCOME_STATUSES`
- Add `"approved_to_ship"` to `getActiveSessions()` query (session is waiting for Runner ship)

### 3. `packages/teamlead/src/bridge/actions.ts` — `approveExecution()`

- **Remove** `ApproveHandler.execute()` call — no more `gh pr merge`
- Change target status from `"approved"` to `"approved_to_ship"`
- **Remove** `onApproved` callback invocation — no merge → no post-merge cleanup
- **Remove** `onApproved` parameter from function signature
- Remove `ApproveHandler` import
- Keep CIPHER recording, update status refs

### 4. `packages/teamlead/src/bridge/actions.ts` — `createActionRouter()`

- Remove `onApproved` parameter from `createActionRouter()`

### 5. `packages/teamlead/src/bridge/plugin.ts`

- Remove `onApproved` callback definition (the `postMergeCleanup` wiring)
- Remove `onApproved` from both `createActionRouter()` calls

### 6. `packages/teamlead/src/bridge/EventFilter.ts`

- Change `session_completed` with `approved` route/status rule: `forum_only` → `notify_agent`
- Priority stays `normal` (Lead should tell Annie in Chat "已 ship")

### 7. `packages/teamlead/src/bridge/dashboard-html.ts`

- Add `.badge-approved_to_ship` CSS class (yellow theme, similar to awaiting_review)

### 8. Files NOT changed (intentionally)

| File | Reason |
|------|--------|
| `ApproveHandler.ts` | Class stays in edge-worker; may be reused by ship action later |
| `event-route.ts` | Auto-approve `running → approved` path unchanged |
| `DirectEventSink.ts` | Same — auto-approve path unchanged |
| `dashboard-data.ts` | `completedToday` counts `completed \|\| approved`; `approved_to_ship` is not yet completed |
| `standup-service.ts` | `COMPLETION_STATUSES` stays `completed \| approved`; `approved_to_ship` not counted |
| `CleanupService.ts` | Cleanup only for truly done sessions; `approved_to_ship` excluded |
| `post-merge.ts` | Still exists, will be wired to Runner ship path later |
| Runner code | Separate task |
