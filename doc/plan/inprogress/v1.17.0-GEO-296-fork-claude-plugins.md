## Step 2: GitHub Actions Sync Workflow

### 2.1 创建 Workflow 文件

在 fork repo 创建 `.github/workflows/sync-upstream.yml`：

```yaml
name: Sync Upstream

on:
  schedule:
    - cron: '0 8 * * *'   # 每日 08:00 UTC (00:00 PST)
  workflow_dispatch:

permissions:
  contents: write
  issues: write

jobs:
  rebase:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v4
        with:
          fetch-depth: 0
          token: ${{ secrets.GITHUB_TOKEN }}

      - name: Configure git
        run: |
          git config user.name "github-actions[bot]"
          git config user.email "41898282+github-actions[bot]@users.noreply.github.com"

      - name: Fetch upstream
        run: |
          git remote add upstream https://github.com/anthropics/claude-plugins-official.git
          git fetch upstream main

      - name: Check for new commits
        id: check
        run: |
          BEHIND=$(git rev-list HEAD..upstream/main --count)
          echo "behind=$BEHIND" >> "$GITHUB_OUTPUT"
          # Save current upstream HEAD before rebase — used later for change detection.
          # After rebase, HEAD~N is unreliable because the base has shifted.
          OLD_UPSTREAM=$(git merge-base HEAD upstream/main)
          echo "old_upstream=$OLD_UPSTREAM" >> "$GITHUB_OUTPUT"

      - name: Rebase
        if: steps.check.outputs.behind != '0'
        id: rebase
        run: |
          if git rebase upstream/main; then
            echo "status=success" >> "$GITHUB_OUTPUT"
          else
            git rebase --abort
            echo "status=conflict" >> "$GITHUB_OUTPUT"
          fi

      - name: Force push
        if: steps.rebase.outputs.status == 'success'
        run: git push origin main --force-with-lease

      - name: Detect discord changes
        if: steps.rebase.outputs.status == 'success'
        id: changes
        run: |
          # Compare upstream range (saved before rebase) to detect discord plugin changes.
          # After rebase, HEAD~N points to new upstream commits, not the old baseline.
          # Use the saved OLD_UPSTREAM_HEAD from before rebase to get the correct range.
          CHANGED=$(git diff --name-only ${{ steps.check.outputs.old_upstream }}..upstream/main -- external_plugins/discord/ || true)
          if [ -n "$CHANGED" ]; then
            echo "discord_changed=true" >> "$GITHUB_OUTPUT"
          else
            echo "discord_changed=false" >> "$GITHUB_OUTPUT"
          fi

      - name: Discord webhook — plugin updated
        if: steps.changes.outputs.discord_changed == 'true'
        run: |
          # IMPORTANT: Send as plain text `content`, NOT embed-only.
          # The Discord plugin only forwards msg.content to Claude Code;
          # embed title/description is NOT passed as message text.
          # Target: #cos-lead-control (NOT #geoforge3d-core) to avoid
          # gate() allowFrom filtering issues with webhook sender identity.
          curl -s -H "Content-Type: application/json" -d '{
            "content": "[Flywheel Fork Sync] Discord Plugin Updated Upstream — fork rebase 后 external_plugins/discord/ 有变化，需要重新部署。请运行 update-discord-plugin。"
          }' "${{ secrets.DISCORD_WEBHOOK_URL }}"

      - name: Notify + issue on conflict
        if: steps.rebase.outputs.status == 'conflict'
        run: |
          curl -s -H "Content-Type: application/json" -d '{
            "content": "[Flywheel Fork Sync] Upstream Rebase CONFLICT — daily rebase 失败，需要手动解决冲突。"
          }' "${{ secrets.DISCORD_WEBHOOK_URL }}"
          gh issue create \
            --title "Upstream rebase conflict — $(date +%Y-%m-%d)" \
            --body "Our patch: single commit adding allowBots to server.ts. Manual rebase needed." \
            --label "upstream-conflict"
        env:
          GH_TOKEN: ${{ secrets.GITHUB_TOKEN }}
```

**注意**: 这个 workflow commit 是 fork 上的第二个 commit（在 allowBots commit 之后）。Rebase 时 GitHub Actions workflow 文件不会跟 upstream 冲突（upstream 没有这个文件路径）。

## Step 3: 本地更新脚本

### 3.1 创建更新脚本

`~/.flywheel/bin/update-discord-plugin.sh`:

```bash
#!/bin/bash
# Update Discord plugin from fork — overwrites Claude Code cache
set -euo pipefail

FORK_REPO="https://github.com/xrliAnnie/claude-plugins-official.git"
FORK_DIR="$HOME/.flywheel/repos/claude-plugins-official"
INSTALLED_PLUGINS="$HOME/.claude/plugins/installed_plugins.json"

# 1. Clone or pull fork
if [ -d "$FORK_DIR/.git" ]; then
  git -C "$FORK_DIR" fetch origin main
  git -C "$FORK_DIR" reset --hard origin/main
else
  mkdir -p "$(dirname "$FORK_DIR")"
  git clone "$FORK_REPO" "$FORK_DIR"
fi

# 2. Resolve active cache directory from installed_plugins.json (authoritative source).
#    Do NOT use `find | sort -V` — orphaned version directories (0.0.1, 0.0.2) exist.
CACHE_DIR=$(python3 -c "
import json, sys
data = json.load(open('$INSTALLED_PLUGINS'))
entries = data.get('plugins', {}).get('discord@claude-plugins-official', [])
if entries: print(entries[0]['installPath'])
else: sys.exit(1)
" 2>/dev/null) || true
if [ -z "$CACHE_DIR" ] || [ ! -d "$CACHE_DIR" ]; then
  echo "ERROR: Discord plugin not found in installed_plugins.json"
  echo "Is the Discord plugin installed? Run: claude plugin install discord"
  exit 1
fi

# 3. Mirror sync to BOTH cache and marketplace directories.
#    Claude Code runs the MCP server from the marketplace dir (not cache).
#    We update both to be safe.
MKT_DIR="$HOME/.claude/plugins/marketplaces/claude-plugins-official/external_plugins/discord"

rsync -a --delete --exclude='node_modules' \
  "$FORK_DIR/external_plugins/discord/" \
  "$CACHE_DIR/"

if [ -d "$MKT_DIR" ]; then
  rsync -a --delete --exclude='node_modules' \
    "$FORK_DIR/external_plugins/discord/" \
    "$MKT_DIR/"
  cd "$MKT_DIR" && bun install --no-summary 2>/dev/null
fi

# 4. Install dependencies in cache
cd "$CACHE_DIR" && bun install --no-summary 2>/dev/null

# 5. Record fork commit SHA for integrity check
FORK_SHA=$(git -C "$FORK_DIR" rev-parse --short HEAD)
echo "$FORK_SHA" > "$CACHE_DIR/.fork-sha"

echo "✓ Discord plugin updated from fork ($FORK_SHA)"
echo "  Cache: $CACHE_DIR"
echo "  Note: Running Leads need restart to pick up changes."
```

### 3.2 创建完整性检查脚本

`~/.flywheel/bin/check-discord-plugin.sh`:

```bash
#!/bin/bash
# Check if Discord plugin cache matches our fork version
# Returns 0 if OK, 1 if needs update
set -euo pipefail

FORK_DIR="$HOME/.flywheel/repos/claude-plugins-official"
INSTALLED_PLUGINS="$HOME/.claude/plugins/installed_plugins.json"

# Resolve from installed_plugins.json (same as update script)
CACHE_DIR=$(python3 -c "
import json, sys
data = json.load(open('$INSTALLED_PLUGINS'))
entries = data.get('plugins', {}).get('discord@claude-plugins-official', [])
if entries: print(entries[0]['installPath'])
else: sys.exit(1)
" 2>/dev/null) || true
if [ -z "$CACHE_DIR" ] || [ ! -d "$CACHE_DIR" ]; then
  echo "MISSING: Discord plugin not in installed_plugins.json"
  exit 1
fi

# Check 1: Content assertion — server.ts must contain allowBots patch.
# Claude Code runs MCP server from MARKETPLACE dir, so check both.
MKT_DIR="$HOME/.claude/plugins/marketplaces/claude-plugins-official/external_plugins/discord"

if ! grep -q 'allowBots' "$CACHE_DIR/server.ts" 2>/dev/null; then
  echo "STALE: server.ts does not contain allowBots in cache"
  exit 1
fi

if [ -d "$MKT_DIR" ] && ! grep -q 'allowBots' "$MKT_DIR/server.ts" 2>/dev/null; then
  echo "STALE: server.ts does not contain allowBots in marketplace (RUNTIME path)"
  exit 1
fi

# Check 2: Fork clone must exist for SHA comparison
if [ ! -d "$FORK_DIR/.git" ]; then
  echo "MISSING: Fork clone not found at $FORK_DIR — run update script first"
  exit 1
fi

# Check 3: SHA marker (fast-path for detecting outdated fork)
if [ ! -f "$CACHE_DIR/.fork-sha" ]; then
  echo "STALE: No fork marker — run update script"
  exit 1
fi

LOCAL_SHA=$(cat "$CACHE_DIR/.fork-sha")
FORK_SHA=$(git -C "$FORK_DIR" rev-parse --short HEAD)
if [ "$LOCAL_SHA" != "$FORK_SHA" ]; then
  echo "OUTDATED: cache=$LOCAL_SHA fork=$FORK_SHA"
  exit 1
fi

echo "OK: Discord plugin matches fork ($LOCAL_SHA), allowBots patch confirmed"
exit 0
```

### 3.3 集成到 claude-lead.sh（必选）

在 `claude-lead.sh` 启动 Lead 前加**强制**完整性检查。这是防止 Claude Code 自动更新覆盖 fork 版本的唯一运行时保障——没有已验证的机制可以禁止 Discord plugin 自动更新。

在 `claude-lead.sh` 的 `# ── Launch Claude with agent identity ──` 之前（约 line 165）插入：

```bash
# ── Discord plugin fork integrity check ─────────────────────
# GEO-296: Ensure Discord plugin is our fork version (with allowBots support).
# Claude Code may overwrite the cache during plugin updates; this preflight
# re-applies our fork if the cache was reverted to the official version.
# Uses absolute paths — do NOT rely on PATH including ~/.flywheel/bin.
FLYWHEEL_BIN="${HOME}/.flywheel/bin"
CHECK_SCRIPT="${FLYWHEEL_BIN}/check-discord-plugin.sh"
UPDATE_SCRIPT="${FLYWHEEL_BIN}/update-discord-plugin.sh"

if [ ! -x "$CHECK_SCRIPT" ] || [ ! -x "$UPDATE_SCRIPT" ]; then
  echo "[lead] ERROR: Discord plugin fork scripts not found or not executable:"
  echo "[lead]   check:  $CHECK_SCRIPT"
  echo "[lead]   update: $UPDATE_SCRIPT"
  echo "[lead] Run GEO-296 setup first. Aborting."
  exit 1
fi

if ! "$CHECK_SCRIPT"; then
  echo "[lead] Discord plugin cache is not fork version, updating..."
  "$UPDATE_SCRIPT"
  # Re-check after update — hard fail if still not matching
  if ! "$CHECK_SCRIPT"; then
    echo "[lead] ERROR: Discord plugin still not fork version after update. Aborting."
    exit 1
  fi
fi
echo "[lead] Discord plugin fork check: OK"
```

**设计要点**：
- 绝对路径调用，不依赖 PATH
- 脚本不存在 → `exit 1`（硬失败）
- 更新后再次检查 → 仍不匹配 → `exit 1`（硬失败）
- 不使用 non-fatal fallback — Lead 不能在没有 fork plugin 的情况下启动

## 实施发现（补充）

### Runtime 路径是 marketplace 目录，不是 cache

实施过程中发现：Claude Code 的 Discord MCP server 实际从 `~/.claude/plugins/marketplaces/claude-plugins-official/external_plugins/discord/` 运行（`bun --cwd` 指向该目录），**不是** `~/.claude/plugins/cache/` 目录。

**影响**：更新脚本必须同时覆盖 marketplace 和 cache 两个目录。已修复 `update-discord-plugin.sh` 和 `check-discord-plugin.sh`。

**验证**：`pgrep -f discord` 显示 bun 进程的 `--cwd` 指向 marketplace 路径，确认 server.ts 从此处加载。
