Current Problem

`core` 里已经有 adapter 抽象，但 `teamlead` 和 `edge-worker` 生产路径仍然直接实例化 Claude 相关实现，导致抽象层没有成为唯一执行入口。

Near-Term Goal

先实现 `Lead = Claude Code`、`Runner = Codex` 的稳定链路，不先追求所有 agent 在能力上完全等价。

Architecture Direction

让 Flywheel 调度的是 role，role 再绑定 adapter，adapter 再包装具体 agent。 这样将来替换 Lead 或 Runner 都不会动 orchestrator 主体。
