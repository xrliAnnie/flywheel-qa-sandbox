# Plan: Lead-Driven Runner Lifecycle — FLY-102 Round 3

**Version**: v1.23.0
**Issue**: FLY-102
**Date**: 2026-04-13
**Source**: `doc/engineer/research/new/FLY-102-lead-driven-runner-lifecycle.md`
**Status**: draft (Round 3)

## 目标

让 Lead 拥有明确、受控的关闭 Runner 能力，触发时机是 **PR merged + tmux cleanup 完成后**。Bridge 继续持有 lifecycle authority；不做自动 done 判定；worktree / docs / MEMORY 更新仍由 Runner / Annie 决定。

> **Round 2 重要澄清**：本次"可关闭"通知仅代表 **tmux cleanup 完成**，不代表 worktree/docs 全部归档。完整 cleanup orchestration（含 doc archive、worktree remove、MEMORY 更新）留给 Runner / Annie / 未来的 executor lifecycle contract，FLY-102 **不**解决。Plan 内不再宣称"full cleanup complete"。

为什么"tmux 已经被 `postMergeTmuxCleanup` 关了"还要 `close_runner`？

1. `postMergeTmuxCleanup` 只覆盖 **ship + PR-merged** 这条业务路径。Annie 中途说「关了 X」/ design / QA 等非 ship 场景仍需 Lead 显式关。
2. `postMergeTmuxCleanup` 失败（lookup miss / kill error）时 Lead 需要 retry 入口。
3. 长期 executor-type 扩展（designer handoff / QA report complete）的 done signal 不是 PR merge，通知面与动作面必须解耦。

双通知的 dedupe / 定位：
- EventFilter 高优 path 在 **业务状态变化时**（`session_completed + completed/approved`）触发，通知 Lead 到 **Lead Chat**，文案是 "ship complete"。
- 新 issue-thread notifier 在 **`postMergeTmuxCleanup` 返回后** 触发，通知落在 **Issue Thread**（FLY-91），文案是 "Runner 完工可关闭"。
- 二者时序先后、channel 不同、文案不同、使用场景不同（前者提醒 Annie 决策，后者给 Lead 下一步 action cue），不做合并。
- 同一 `execution_id` 的 thread notification 通过 audit event `runner_ready_to_close_notified` 做幂等（见 Step 3），避免双路径都触发时发两条。

```ts
// packages/teamlead/src/bridge/post-ship-finalization.ts
import { postMergeTmuxCleanup } from "./post-merge.js";
import { emitRunnerReadyToCloseNotification } from "./runner-ready-to-close-notifier.js";
import { archiveChatThread, removeUserFromChatThread } from "./chat-thread-utils.js";
import { resolveLeadForIssue } from "../ProjectConfig.js";
import type { StateStore } from "../StateStore.js";
import type { ProjectsConfig } from "../ProjectConfig.js";

/**
 * Shared predicate — aligns with event-route.ts:319-341.
 * Must hold BOTH for DES path and event-route path before tmux cleanup / notifier fire.
 */
export function isPostApproveShipComplete(args: {
  existingStatus: string | undefined;      // session.status BEFORE this event applied
  route: string | undefined;
  landingStatus: { status?: string } | undefined;
}): boolean {
  if (args.existingStatus === "approved_to_ship") return true;         // FLY-58 post-approve ship
  if (args.route === "auto_approve" && args.landingStatus?.status === "merged") return true;
  return false;
}

export interface PostShipOpts {
  executionId: string;
  issueId: string;
  issueIdentifier?: string;
  projectName: string;
  sessionStatus: string;                  // final status written by DES / event-route
  discordOwnerUserId?: string;
  fallbackBotToken?: string;
}

/**
 * Serialized post-ship orchestrator. Guarantees ordering:
 *   1. postMergeTmuxCleanup — close Runner tmux window + audit
 *   2. emitRunnerReadyToCloseNotification — post "🏁 Runner 完工可关闭" into Issue Thread
 *   3. archiveChatThread + removeOwnerFromChatThread — clear Annie's sidebar
 * Fire-and-forget at call site; returns void. All stages internally swallow & audit errors.
 */
export async function runPostShipFinalization(
  opts: PostShipOpts,
  deps: { store: StateStore; projects: ProjectsConfig },
): Promise<void> {
  const { store, projects } = deps;

  // (1) tmux cleanup — idempotent via killTmuxWindow. Returns PostMergeResult
  //     with fields { tmuxClosed, errors } (existing contract; NOT renamed).
  const cleanup = await postMergeTmuxCleanup(
    { executionId: opts.executionId, issueId: opts.issueId, projectName: opts.projectName },
    store,
  ).catch((err) => {
    console.error(`[post-ship] postMergeTmuxCleanup failed:`, (err as Error).message);
    return { tmuxClosed: false, errors: [(err as Error).message] };
  });

  // Resolve lead + thread ONCE, reused by both notifier and archiver.
  const labels = store.getSessionLabels(opts.executionId);
  const { lead } = resolveLeadForIssue(projects, opts.projectName, labels);
  const chatChannel = lead.chatChannel;
  const botToken = lead.botToken ?? opts.fallbackBotToken;
  const thread = chatChannel ? store.getChatThreadByIssue(opts.issueId, chatChannel) : undefined;

  // (2) notifier — atomic dedupe; must run BEFORE archive so message lands in open thread.
  await emitRunnerReadyToCloseNotification(
    {
      executionId: opts.executionId,
      issueId: opts.issueId,
      issueIdentifier: opts.issueIdentifier,
      projectName: opts.projectName,
      sessionStatus: opts.sessionStatus,
      tmuxClosed: cleanup.tmuxClosed,
      errors: cleanup.errors?.length ? cleanup.errors : undefined,
      thread,
      botToken,
    },
    { store },
  );

  // (3a) remove Annie from thread (FLY-91 parity with previous DES behavior),
  //       and (3b) archive — both AFTER notifier so the "🏁 可关闭" post lands
  //       in the still-open thread and Annie still sees it in her sidebar
  //       until Discord unsubscribes her. Discord auto-unarchives on reply.
  if (thread && botToken) {
    if (opts.discordOwnerUserId) {
      removeUserFromChatThread(thread.thread_id, opts.discordOwnerUserId, botToken).catch(
        (err) => console.warn(`[post-ship] removeUserFromChatThread failed:`, (err as Error).message),
      );
    }
    archiveChatThread(thread.thread_id, botToken).catch((err) =>
      console.warn(`[post-ship] archiveChatThread failed:`, (err as Error).message),
    );
  }
}
```

### Step 2: Bridge `close-runner` endpoint + 幂等 close policy

**新文件**：`packages/teamlead/src/bridge/close-runner.ts`

**职责**：
1. 校验 session 存在 + 在 Lead scope 内
2. Close eligibility guard（见下 §Close Policy）
3. 调 `killTmuxWindow`（复用 `tmux-lookup.ts`）
4. 写 audit event `lead_close_runner` / `lead_close_runner_blocked` / `lead_close_runner_failed`

#### Close Policy

`CLOSE_ELIGIBLE_STATES` 基于 "Runner 不再需要 tmux" 原则，囊括所有非运行态 outcome：

```ts
// Any status where Runner is no longer doing work and tmux is safe to reap.
// Rationale: rejected / deferred / shelved / terminated are all non-running
// terminal outcomes — keeping tmux alive serves no purpose. This aligns with
// OUTCOME_STATUSES in StateStore.ts:5-16 minus approved/approved_to_ship
// (those still need tmux for Runner to finish the ship step).
export const CLOSE_ELIGIBLE_STATES: ReadonlySet<string> = new Set([
  "completed",
  "failed",
  "blocked",
  "rejected",
  "deferred",
  "shelved",
  "terminated",
]);
```

**明确排除**：`running`, `awaiting_review`, `approved`, `approved_to_ship` — 这些态 Runner 还可能需要 tmux 继续动作或继续被人观察。

#### `closeRunner()` 实现

```ts
// packages/teamlead/src/bridge/close-runner.ts
import type { StateStore } from "../StateStore.js";
import { getTmuxTargetFromCommDb, killTmuxWindow } from "./tmux-lookup.js";

export const CLOSE_ELIGIBLE_STATES: ReadonlySet<string> = new Set([
  "completed", "failed", "blocked",
  "rejected", "deferred", "shelved", "terminated",
]);

export interface CloseRunnerOpts {
  executionId: string;
  issueId: string;
  projectName: string;
  reason?: string;
  leadId?: string;
  /**
   * Reserved for future per-executor dispatch. NOT exposed via MCP tool.
   * See doc/engineer/research/new/FLY-102-lead-driven-runner-lifecycle.md §4.2
   */
  executorType?: string;
}

export interface CloseRunnerResult {
  closed: boolean;           // tmux window ended up not-running (killed or already gone)
  alreadyGone?: boolean;     // target not found in CommDB or window missing on tmux
  error?: string;            // only set for true failure (e.g. status_not_eligible, kill_error)
}

export async function closeRunner(
  opts: CloseRunnerOpts,
  store: StateStore,
): Promise<CloseRunnerResult> {
  const session = store.getSession(opts.executionId);
  if (!session) {
    return { closed: false, error: "session_not_found" };
  }

  if (!CLOSE_ELIGIBLE_STATES.has(session.status)) {
    const err = `status_not_eligible:${session.status}`;
    store.insertEvent({
      event_id: `close-runner-blocked-${opts.executionId}-${Date.now()}`,
      execution_id: opts.executionId,
      issue_id: opts.issueId,
      project_name: opts.projectName,
      event_type: "lead_close_runner_blocked",
      source: "bridge.close-runner",
      payload: {
        status: session.status,
        eligibleStates: Array.from(CLOSE_ELIGIBLE_STATES),
        reason: opts.reason,
        leadId: opts.leadId,
      },
    });
    return { closed: false, error: err };
  }

  const target = getTmuxTargetFromCommDb(opts.executionId, opts.projectName);

  // No target = tmux already gone = idempotent success.
  if (!target) {
    store.insertEvent({
      event_id: `close-runner-${opts.executionId}-${Date.now()}`,
      execution_id: opts.executionId,
      issue_id: opts.issueId,
      project_name: opts.projectName,
      event_type: "lead_close_runner",
      source: "bridge.close-runner",
      payload: {
        closed: true,
        alreadyGone: true,
        reason: opts.reason,
        leadId: opts.leadId,
        executorType: opts.executorType ?? "engineer",
      },
    });
    return { closed: true, alreadyGone: true };
  }

  const res = await killTmuxWindow(target.tmuxWindow);
  // killTmuxWindow treats "window missing" as success; res.error is only a real failure.
  const eventType = res.error ? "lead_close_runner_failed" : "lead_close_runner";
  store.insertEvent({
    event_id: `close-runner-${opts.executionId}-${Date.now()}`,
    execution_id: opts.executionId,
    issue_id: opts.issueId,
    project_name: opts.projectName,
    event_type: eventType,
    source: "bridge.close-runner",
    payload: {
      closed: res.killed,
      reason: opts.reason,
      leadId: opts.leadId,
      executorType: opts.executorType ?? "engineer",
      tmuxError: res.error,
    },
  });

  return { closed: res.killed, error: res.error };
}
```

#### Endpoint wiring (`plugin.ts`, 紧贴现有 `close-tmux` endpoint)

```ts
app.post(
  "/api/sessions/:executionId/close-runner",
  tokenAuthMiddleware(config.apiToken),
  async (req, res) => {
    const executionId = req.params.executionId as string;
    const { leadId, reason, executorType } = (req.body ?? {}) as {
      leadId?: string;
      reason?: string;
      executorType?: string;
    };

    const session = store.getSession(executionId);
    if (!session) {
      res.status(404).json({ error: "Session not found" });
      return;
    }

    if (leadId && projects && !matchesLead(session, leadId, projects)) {
      res.status(403).json({
        success: false,
        message: `Session ${executionId} is outside lead "${leadId}" scope`,
      });
      return;
    }

    const result = await closeRunner(
      { executionId, issueId: session.issue_id, projectName: session.project_name,
        reason, leadId, executorType },
      store,
    );

    if (!result.closed && result.error?.startsWith("status_not_eligible:")) {
      res.status(409).json({
        success: false,
        message: `Cannot close runner: ${result.error}. Eligible states: ${Array.from(CLOSE_ELIGIBLE_STATES).join(", ")}.`,
      });
      return;
    }

    res.json({
      success: result.closed,
      alreadyGone: result.alreadyGone ?? false,
      error: result.error,
    });
  },
```

### Step 3: Notifier — `emitRunnerReadyToCloseNotification`（原子幂等 + 真实签名）

**新文件**：`packages/teamlead/src/bridge/runner-ready-to-close-notifier.ts`

**Round 3 关键修正**：
- 输入不再内部 resolve lead/thread/token — 改为由 `runPostShipFinalization` 预解析后传入（避免重复做同样的 `getSessionLabels + resolveLeadForIssue`，并确保 notifier 和 archive 对同一个 thread 操作）。
- **原子幂等**：使用稳定 `event_id = runner-ready-to-close-${executionId}`，`insertEvent` 抢占。`insertEvent` 在 UNIQUE 冲突时返回 `false`（`StateStore.ts:420-449` 已确认），第一个调用抢到锁发消息，其余 call site 同时进来会直接 short-circuit。**不再**先 `findEvents` 再插入（此做法有 TOCTOU race）。
- 失败 event 用不同的 `event_id`（`...-failed-${randomUUID()}`）避免和成功锁冲突。

```ts
// packages/teamlead/src/bridge/runner-ready-to-close-notifier.ts
import { randomUUID } from "node:crypto";
import type { StateStore } from "../StateStore.js";

// Avoid adding a new StateStore export — derive the shape from the accessor.
// getChatThreadByIssue returns { thread_id, channel_id } | undefined.
export type ChatThreadRef = NonNullable<ReturnType<StateStore["getChatThreadByIssue"]>>;

export interface ReadyToCloseOpts {
  executionId: string;
  issueId: string;
  issueIdentifier?: string;
  projectName: string;
  sessionStatus: string;
  tmuxClosed: boolean;
  errors?: string[];
  thread?: ChatThreadRef;     // resolved by runPostShipFinalization
  botToken?: string;          // lead.botToken ?? config.discordBotToken
}

export interface ReadyToCloseDeps {
  store: StateStore;
  fetchImpl?: typeof fetch;   // test seam
}

const DISCORD_API = "https://discord.com/api/v10";

export async function emitRunnerReadyToCloseNotification(
  opts: ReadyToCloseOpts,
  deps: ReadyToCloseDeps,
): Promise<void> {
  const { store, fetchImpl = fetch } = deps;

  // ── (A) ATOMIC CLAIM ──────────────────────────────────────────
  // Stable event_id = dedupe key. UNIQUE constraint on event_id (StateStore.ts:420-449)
  // guarantees at-most-one insert wins when DES + event-route fire concurrently.
  const claimId = `runner-ready-to-close-${opts.executionId}`;
  const claimed = store.insertEvent({
    event_id: claimId,
    execution_id: opts.executionId,
    issue_id: opts.issueId,
    project_name: opts.projectName,
    event_type: "runner_ready_to_close_claim",   // intent, not yet notified
    source: "bridge.ready-to-close-notifier",
    payload: { claimedAt: new Date().toISOString() },
  });
  if (!claimed) return;   // another caller already claimed → we are the no-op path

  // ── (B) VALIDATE INPUTS ──────────────────────────────────────
  const thread = opts.thread;
  if (!thread?.thread_id) {
    store.insertEvent({
      event_id: `runner-ready-to-close-skipped-${randomUUID()}`,
      execution_id: opts.executionId,
      issue_id: opts.issueId,
      project_name: opts.projectName,
      event_type: "runner_ready_to_close_skipped",
      source: "bridge.ready-to-close-notifier",
      payload: { reason: "no_chat_thread", claimId },
    });
    return;
  }
  if (!opts.botToken) {
    store.insertEvent({
      event_id: `runner-ready-to-close-skipped-${randomUUID()}`,
      execution_id: opts.executionId,
      issue_id: opts.issueId,
      project_name: opts.projectName,
      event_type: "runner_ready_to_close_skipped",
      source: "bridge.ready-to-close-notifier",
      payload: { reason: "no_bot_token", claimId },
    });
    return;
  }

  // ── (C) COMPOSE + POST ───────────────────────────────────────
  const errLine = opts.errors?.length
    ? `\n- Cleanup errors: ${opts.errors.join("; ")}`
    : "";
  const body = [
    `🏁 **Runner 完工可关闭** — ${opts.issueIdentifier ?? opts.issueId}`,
    ``,
    `- Execution: \`${opts.executionId}\``,
    `- Session status: \`${opts.sessionStatus}\``,
    `- Tmux: ${opts.tmuxClosed ? "已由 postMergeTmuxCleanup 关闭" : "仍在（Lead 可用 close_runner 收尾）"}${errLine}`,
    ``,
    `Lead：按规则决策（默认向 Annie 确认后再调 \`close_runner\`）。`,
  ].join("\n");

  try {
    const resp = await fetchImpl(
      `${DISCORD_API}/channels/${thread.thread_id}/messages`,
      {
        method: "POST",
        headers: {
          Authorization: `Bot ${opts.botToken}`,
          "Content-Type": "application/json",
        },
        body: JSON.stringify({ content: body }),
      },
    );
    if (!resp.ok) {
      const errText = await resp.text().catch(() => "");
      store.insertEvent({
        event_id: `runner-ready-to-close-failed-${randomUUID()}`,
        execution_id: opts.executionId,
        issue_id: opts.issueId,
        project_name: opts.projectName,
        event_type: "runner_ready_to_close_notify_failed",
        source: "bridge.ready-to-close-notifier",
        payload: { status: resp.status, body: errText.slice(0, 500), claimId },
      });
      return;
    }
  } catch (err) {
    store.insertEvent({
      event_id: `runner-ready-to-close-failed-${randomUUID()}`,
      execution_id: opts.executionId,
      issue_id: opts.issueId,
      project_name: opts.projectName,
      event_type: "runner_ready_to_close_notify_failed",
      source: "bridge.ready-to-close-notifier",
      payload: { error: (err as Error).message, claimId },
    });
    return;
  }

  // ── (D) SUCCESS AUDIT ────────────────────────────────────────
  store.insertEvent({
    event_id: `runner-ready-to-close-notified-${randomUUID()}`,
    execution_id: opts.executionId,
    issue_id: opts.issueId,
    project_name: opts.projectName,
    event_type: "runner_ready_to_close_notified",
    source: "bridge.ready-to-close-notifier",
    payload: { threadId: thread.thread_id, tmuxClosed: opts.tmuxClosed, claimId },
  });
}
```

**terminal-mcp tool 定义**：
```ts
server.tool(
  "close_runner",
  [
    "Close a Runner's tmux window after it has reached a non-running outcome.",
    "Eligible statuses: completed, failed, blocked, rejected, deferred, shelved, terminated.",
    "REJECTS: running, awaiting_review, approved, approved_to_ship — those must be approved/rejected first.",
    "Use after Annie confirms closure, or per team-lead pre-authorized rules.",
    "Idempotent: if the tmux window is already gone, returns success.",
  ].join(" "),
  {
    issue_identifier: z.string().optional()
      .describe("Linear issue identifier (e.g. 'FLY-102'). Either this or execution_id required."),
    execution_id: z.string().optional()
      .describe("CommDB execution_id. Either this or issue_identifier required."),
    reason: z.string()
      .describe("Why closing (for audit trail — e.g. 'Annie approved after ship complete')."),
  },
  async ({ issue_identifier, execution_id, reason }) => {
    if (!issue_identifier && !execution_id) {
      return { content: [{ type: "text" as const, text: "Error: provide issue_identifier or execution_id" }], isError: true };
    }
    const bridgeUrl = process.env.BRIDGE_URL;
    const token = process.env.TEAMLEAD_API_TOKEN;
    if (!bridgeUrl || !token) {
      return { content: [{ type: "text" as const, text: "Error: BRIDGE_URL / TEAMLEAD_API_TOKEN not configured" }], isError: true };
    }

    // Resolve execution_id via Bridge by_identifier endpoint when only identifier given.
    let resolvedExec = execution_id;
    if (!resolvedExec && issue_identifier) {
      const r = await fetch(
        `${bridgeUrl}/api/sessions?mode=by_identifier&identifier=${encodeURIComponent(issue_identifier)}`,
        { headers: { Authorization: `Bearer ${token}` } },
      );
      if (!r.ok) {
        return { content: [{ type: "text" as const, text: `Error: lookup failed (${r.status})` }], isError: true };
      }
      const json = await r.json() as { sessions: Array<{ execution_id: string; status: string }> };
      if (!json.sessions?.length) {
        return { content: [{ type: "text" as const, text: `Error: no session for ${issue_identifier}` }], isError: true };
      }
      resolvedExec = json.sessions[0].execution_id;
    }

    // Scope guard (local) — reuse getSessionScoped so MCP-layer errors are consistent.
    try {
      const db = openDb();
      try {
        getSessionScoped(db, resolvedExec!, { requireExactLead: true });
      } finally {
        db.close();
      }
    } catch (e) {
      return { content: [{ type: "text" as const, text: `Error: ${(e as Error).message}` }], isError: true };
    }

    const r = await fetch(`${bridgeUrl}/api/sessions/${resolvedExec}/close-runner`, {
      method: "POST",
      headers: { Authorization: `Bearer ${token}`, "Content-Type": "application/json" },
      body: JSON.stringify({ leadId, reason }),
    });
    const body = await r.text();
    if (r.status === 409) {
      return { content: [{ type: "text" as const, text: `Refused: ${body}` }] };
    }
    if (!r.ok) {
      return { content: [{ type: "text" as const, text: `Error (${r.status}): ${body}` }], isError: true };
    }
    return { content: [{ type: "text" as const, text: body }] };
  },
);
```
