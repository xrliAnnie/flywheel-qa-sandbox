# Plan: v0.2 Step 2a — Core Integration Wiring

## Context

Step 1 (PR #5, merged) delivered 5 independent infrastructure components:
- **Semaphore** / **ProjectLock** (`packages/core/`)
- **HookCallbackServer** / **WorktreeManager** / **SkillInjector** (`packages/edge-worker/`)

Step 2 connects these components into an end-to-end single-issue flow. Due to scope (~1,010 LOC total), Step 2 is split into three sub-PRs:

|----------|-------|-------------|
| **2a** Core Integration (this plan) | TmuxRunner callback + Blueprint worktree/skill wiring + EvidenceCollector | Step 1 only |
| **2b** Decision Layer | HardRuleEngine + HaikuTriage + HaikuVerifier + Fallback + AuditLogger | Step 2a |

**Architecture doc**: `doc/architecture/v0.2-architecture.md`
**Step 1 plan**: `doc/engineer/plan/draft/v0.2-step1-infrastructure.md`

---

## Step 2a Goal

> Single Linear issue → worktree → skill injection → Claude Code session (HTTP callback completion) → evidence collection → BlueprintResult with evidence.

Step 2a does NOT add decision logic or Slack notifications — it wires the execution pipeline so Step 2b can plug in a DecisionLayer.

**Scope: single-repo projects only.** Step 2a's worktree integration assumes a single git repository. Multi-repo projects (e.g., GeoForge3D with umbrella dir + independent sub-repos) are NOT supported by `git worktree add` — child repos don't appear in the worktree. Multi-repo worktree support (`.worktreeinclude` / sub-repo cloning) is deferred to a follow-up PR. For multi-repo projects, Blueprint falls back to the v0.1.1 path (no worktree, `worktreeManager` not injected).

### Target State

Two completion modes depending on whether `hookServer` is injected:

- **v0.2 mode** (`hookServer` present): HTTP callback (primary) + pane_dead poller (fallback). Marker file path disabled — per architecture doc, v0.2 completion is "HTTP callback + pane_dead" only. The shared marker dir (`/tmp/flywheel/sessions`) is unsafe for parallel and no longer needed when HTTP callback provides per-token correlation.
- **v0.1.1 mode** (`hookServer` absent): Existing marker file + pane_dead behavior. Fully backward compatible.

### Changes to `waitForCompletion()`

Two-mode completion detection:

```typescript
private async waitForCompletion(
  claudeSessionId: string,
  windowId: string,
  timeoutMs: number,
  callbackToken?: string,  // NEW
): Promise<boolean> {
  return new Promise((resolve) => {
    let settled = false;
    const settle = (timedOut: boolean) => {
      if (settled) return;
      settled = true;
      // cleanup: clear timer, unwatch, stop poller
      resolve(timedOut);
    };

    // Timeout
    const timer = setTimeout(() => settle(true), timeoutMs);

    if (this.hookServer && callbackToken) {
      // v0.2 mode: HTTP callback (primary) + pane_dead poller (fallback)
      // NO marker file watcher — markers are not per-token safe for parallel

      // Path 1: HTTP callback
      this.hookServer.waitForCompletion(callbackToken, timeoutMs)
        .then((event) => {
          if (event) settle(false);
        });

      // Path 2: pane_dead poller (fallback — races with callback)
      const poller = setInterval(() => {
        try {
          const result = this.execFileFn("tmux", [
            "list-panes", "-t", windowId, "-F", "#{pane_dead}",
          ]);
          if (result.stdout.trim() === "1") settle(false);
        } catch {
          // Window gone → pane is dead
          settle(false);
        }
      }, this.pollIntervalMs);
      // Store poller for cleanup in settle()
    } else {
      // v0.1.1 mode: marker file watcher + pane_dead poller (existing behavior)
      // ... existing fs.watch + readdirSync + pane_dead logic unchanged ...
    }
  });
}
```

### Key Design Decisions

1. **hookServer is optional** — v0.1.1 behavior preserved when undefined
2. **Per-window env injection** — `tmux new-window -e` is per-window, not `set-environment` (which is session-scoped and would break parallel windows)
3. **callbackToken is the correlation key** — NOT `sessionId` (which is Claude's internal ID and doesn't match our UUID)
4. **Two-mode completion** — with hookServer: HTTP callback + pane_dead (no marker). Without: marker + pane_dead (v0.1.1). Per architecture doc, v0.2 drops markers for parallel safety.
5. **FlywheelRunRequest extended** — add optional `issueId?: string` field for callback. Port/token are TmuxRunner internals, but `issueId` flows through the run request because TmuxRunner needs it for the env var. This is a minor interface extension (optional field, backward compat).
6. **label vs issueId** — `request.label` is for tmux window name (human-readable, may contain spaces). `request.issueId` is for callback URL (URL-safe, e.g., "GEO-95"). They are different fields for different purposes.

```typescript
export interface ExecutionEvidence {
  // From GitCheckResult (required — throw on failure)
  commitCount: number;
  filesChangedCount: number;
  commitMessages: string[];

  // Extended evidence (best-effort — degrade gracefully)
  changedFilePaths: string[];     // git diff --name-only baseSha..HEAD
  linesAdded: number;             // parsed from git diff --stat
  linesRemoved: number;           // parsed from git diff --stat
  diffSummary: string;            // git diff baseSha..HEAD | head -c 2000
  headSha: string | null;         // git rev-parse HEAD

  // Metadata
  partial: boolean;               // true if any best-effort field degraded
  durationMs: number;             // passed in from runner result
}
```

### Implementation

```typescript
async collect(cwd, baseSha, gitResult, durationMs): Promise<ExecutionEvidence> {
  let partial = false;

  // Required fields — from GitCheckResult (already validated)
  const { commitCount, filesChanged: filesChangedCount, commitMessages } = gitResult;

  // Best-effort fields — each wrapped in try/catch
  const changedFilePaths = await this.getChangedFiles(cwd, baseSha).catch(() => {
    partial = true;
    return [];
  });

  const { added, removed } = await this.getDiffStats(cwd, baseSha).catch(() => {
    partial = true;
    return { added: 0, removed: 0 };
  });

  const diffSummary = await this.getDiffSummary(cwd, baseSha).catch(() => {
    partial = true;
    return "";
  });

  const headSha = await this.getHeadSha(cwd).catch(() => {
    partial = true;
    return null;
  });

  return {
    commitCount, filesChangedCount, commitMessages,
    changedFilePaths, linesAdded: added, linesRemoved: removed,
    diffSummary, headSha, partial, durationMs,
  };
}
```

### Git Commands

| Method | Command | Parse |
|--------|---------|-------|
| `getChangedFiles` | `git diff --name-only {baseSha}..HEAD` | Split by newline, filter empty |
| `getDiffStats` | `git diff --numstat {baseSha}..HEAD` | Sum columns 1 (added) and 2 (removed) |
| `getDiffSummary` | `git diff {baseSha}..HEAD` | Truncate to 2000 chars |
| `getHeadSha` | `git rev-parse HEAD` | Trim |

### Why `--numstat` Instead of `--stat`

`--stat` output format varies by terminal width and is hard to parse reliably. `--numstat` outputs machine-readable `<added>\t<removed>\t<file>` per line. Binary files show `-\t-\t<file>` — these are counted as 0.

### WorktreeManager Extension: `removeIfExists()`

Blueprint needs a safe rerun cleanup API that keeps path construction internal to WorktreeManager and also handles the local branch (which `remove()` alone does not delete).

```typescript
// Add to WorktreeManager class:
async removeIfExists(
  mainRepoPath: string,
  projectName: string,
  issueId: string,
): Promise<boolean> {
  const branch = `flywheel-${issueId}`;
  const worktreePath = path.join(this.baseDir, projectName, branch);

  // Step 1: remove worktree if registered
  if (await this.isRegistered(mainRepoPath, worktreePath)) {
    await this.remove(mainRepoPath, worktreePath);
  }

  // Step 2: delete local branch if it still exists
  // git worktree prune does NOT delete the branch — only the worktree registration.
  // Without this, next create() with -b flywheel-{issueId} fails:
  //   "fatal: a branch named 'flywheel-GEO-42' already exists"
  try {
    await this.exec(
      "git", ["-C", mainRepoPath, "branch", "-D", branch], mainRepoPath,
    );
    return true;
  } catch (err: unknown) {
    // Only swallow "branch not found" — all other git errors propagate
    const msg = err instanceof Error ? err.message : String(err);
    if (!msg.includes("not found")) throw err;
    return false;  // Branch didn't exist — benign (first run or already cleaned)
  }
}
```

**Why `branch -D` and not `-d`**: The worktree branch may have un-merged commits (e.g., from a failed run). `-d` would refuse; `-D` force-deletes. This is safe because:
- The branch is a flywheel-managed temp branch, not a user branch
- If the PR was already created, the remote branch survives (only local ref is deleted)
- Re-create will start fresh from `origin/main`

### Updated `run()` Flow

```typescript
async run(node, projectRoot, ctx): Promise<BlueprintResult> {
  let cwd = projectRoot;
  let worktreeInfo: WorktreeInfo | undefined;

  // ── Worktree setup (NEW — own try/catch) ──────────────────
  // Worktree errors → BlueprintResult.error (recoverable).
  // Kept SEPARATE from main flow to preserve existing throw semantics
  // for assertCleanTree/captureBaseline (infra errors that DagDispatcher
  // and existing tests expect to propagate as thrown exceptions).
  if (this.worktreeManager) {
    const projectName = ctx.projectName ?? ctx.teamName;  // fallback for compat
    try {
      await this.worktreeManager.removeIfExists(projectRoot, projectName, node.id);
      worktreeInfo = await this.worktreeManager.create({
        mainRepoPath: projectRoot,
        projectName,
        issueId: node.id,
      });
      cwd = worktreeInfo.worktreePath;
    } catch (error) {
      return {
        success: false,
        error: error instanceof Error ? error.message : String(error),
        worktreePath: worktreeInfo?.worktreePath,
      };
    }
  }

  // ── Git preflight (existing — THROWS on failure) ──────────
  // assertCleanTree: dirty tree → throw (existing contract)
  // captureBaseline: git error → throw (existing contract)
  // DagDispatcher catches these at a higher level.
  await this.gitChecker.assertCleanTree(cwd);
  const baseSha = await this.gitChecker.captureBaseline(cwd);

  // ── Pre-Hydrate (existing) ────────────────────────────────
  const hydratedCtx = await this.hydrator.hydrate(node);

  // ── Skill injection (NEW — best-effort, non-blocking) ─────
  // Per architecture doc: skill injection failure → log warning, continue.
  // Skills enhance session quality but are NOT required for the session to
  // produce commits. Failing here should not abort the entire run.
  if (this.skillInjector) {
    const projectName = ctx.projectName ?? ctx.teamName;
    try {
      await this.skillInjector.inject(cwd, {
        issueId: hydratedCtx.issueId,
        issueTitle: hydratedCtx.issueTitle,
        issueDescription: hydratedCtx.issueDescription,
        projectName,
        testCommand: this.skillsConfig?.test_command,
        lintCommand: this.skillsConfig?.lint_command,
        buildCommand: this.skillsConfig?.build_command,
        testFramework: this.skillsConfig?.test_framework,
      });
    } catch (err) {
      console.warn(`[Blueprint] Skill injection failed (non-fatal): ${
        err instanceof Error ? err.message : String(err)
      }`);
    }
  }

  // ── Runner execution (existing — catch runner errors) ─────
  let result: FlywheelRunResult;
  try {
    const runner = this.getRunner(ctx.runnerName);
    const timeoutMs = ctx.sessionTimeoutMs ?? 1_800_000;  // default 30 min
    result = await runner.run({
      prompt, cwd,
      label: `${hydratedCtx.issueId}-${hydratedCtx.issueTitle}`,  // human-readable tmux window name
      issueId: hydratedCtx.issueId,  // URL-safe ID for callback env var
      permissionMode: "bypassPermissions",
      appendSystemPrompt,
      timeoutMs,
    });
  } catch (err) {
    const errorMsg = err instanceof Error ? err.message : String(err);
    return {
      success: false,
      durationMs: Date.now() - startTime,
      error: errorMsg,
      worktreePath: worktreeInfo?.worktreePath,
    };
  }

  // ── Git result check (existing — THROWS on infra error) ───
  // Let git check infra errors propagate (existing contract).
  const gitResult = await this.gitChecker.check(cwd, baseSha);

  // ── Evidence collection (NEW — conditional) ───────────────
  let evidence: ExecutionEvidence | undefined;
  if (this.evidenceCollector) {
    evidence = await this.evidenceCollector.collect(
      cwd, baseSha, gitResult, result.durationMs ?? 0,
    );
  }

  const success = gitResult.commitCount > 0 && !result.timedOut;

  // ── Window lifecycle (existing) ───────────────────────────
  if (success && result.tmuxWindow) {
    await this.killTmuxWindow(result.tmuxWindow);
  }

  return {
    success,
    costUsd: result.costUsd,
    sessionId: result.sessionId,
    tmuxWindow: success ? undefined : result.tmuxWindow,
    durationMs: result.durationMs,
    worktreePath: worktreeInfo?.worktreePath,
    evidence,
  };
  // NOTE: No worktree cleanup in Step 2a.
  // Step 2b adds decision-driven cleanup policy:
  //   auto_approve + merged → delete worktree
  //   needs_review → keep (CEO reviewing)
  //   blocked → keep (TTL 24h)
}
```

### Error Contract Summary

| Step | Error behavior | Why |
|------|---------------|-----|
| Worktree create | → `BlueprintResult.error` | Recoverable — worktree preserved for debugging |
| assertCleanTree | → **throw** | Existing contract — DagDispatcher/tests expect this |
| captureBaseline | → **throw** | Existing contract — infra error |
| Skill injection | → **warn + continue** | Best-effort enhancement — session runs without skills |
| Runner | → `BlueprintResult.error` | Existing pattern — runner errors are expected |
| gitChecker.check | → **throw** | Existing contract — infra error propagates |
| Evidence collection | Best-effort (internal `partial` flag) | Never throws |

### Worktree Cleanup Policy (Step 2a)

Step 2a does NOT delete worktrees after execution. Cleanup is deferred to Step 2b where the Decision Layer determines the outcome:
- `auto_approve` + merged → delete immediately
- `needs_review` → keep (CEO is reviewing the PR)
- `blocked` → keep with TTL

For Step 2a, worktrees accumulate and can be manually pruned via `WorktreeManager.pruneOrphans()`.

## Implementation Notes

1. **ESM imports**: use `.js` extension (same as Step 1)
2. **Optional constructor params**: Blueprint v0.2 deps are optional — trailing `?` params, not a config object. This preserves the existing constructor signature (no breaking change).
3. **No DagDispatcher changes**: Step 2a doesn't touch DagDispatcher. The v0.2 deps are wired at the composition root (`scripts/run-issue.ts` in Task 7). DagDispatcher's webhook-based wiring is Step 2b/2c scope.
4. **TmuxRunner backward compat**: `hookServer?: IHookCallbackServer` is optional. All existing tests pass without it.
5. **tmux -e for env vars**: Per-window injection, NOT `set-environment` (session-scoped). Critical for future parallel execution — each window needs its own token.
6. **WorktreeManager is NOT injected as interface**: Unlike HookCallbackServer (cross-package), WorktreeManager stays in edge-worker and Blueprint uses it directly. No interface extraction needed.
7. **ExecutionEvidenceCollector reuses `GitExecFileFn`**: Imports `GitExecFileFn` from `GitResultChecker.ts` (already re-exported from `packages/edge-worker/src/index.ts:29`). No parallel type definition — prevents type drift.
8. **`--numstat` not `--stat`**: Machine-readable output for reliable line count parsing.
9. **Evidence `durationMs`**: Passed from `FlywheelRunResult.durationMs`, not measured by collector.
10. **No worktree cleanup in Step 2a**: Intentional — cleanup policy depends on Decision Layer outcome (Step 2b).
11. **Worktree rerun safety**: Blueprint calls `WorktreeManager.removeIfExists(mainRepoPath, projectName, issueId)` before `create()`. This method: (a) removes the worktree registration + dir via `remove()`, (b) force-deletes the local `flywheel-{issueId}` branch via `git branch -D`. Both steps are needed because `git worktree prune` does NOT delete the local branch ref — without step (b), the next `create()` with `-b flywheel-{issueId}` would fail with "branch already exists".
12. **`projectName` source**: `BlueprintContext.projectName` (from FlywheelConfig or directory basename). Falls back to `ctx.teamName` for backward compat. Never guess from teamName — they're different concepts.
13. **`sessionTimeoutMs` source**: `BlueprintContext.sessionTimeoutMs` (from `ParallelConfig.session_timeout_minutes * 60_000`). Falls back to 30 min default. Composition root (Task 7) owns the conversion.
