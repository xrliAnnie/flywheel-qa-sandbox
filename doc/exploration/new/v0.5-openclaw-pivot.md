## 0. Executive Summary

从自建 TeamLead Brain 转向 OpenClaw multi-agent 架构。核心动机：CEO 需要一个**持久的、有记忆的、能使用工具的** AI 中枢大脑，而不是每次问题都 spawn 一个无状态的 `claude -p` 进程。

OpenClaw 提供：持久 session（JSONL）、tool use loop、memory、heartbeat、sub-agent、multi-channel gateway。

**关键风险**：`claude-max-api-proxy` 不支持 tool_use。需要验证 `setup-token` 认证是否支持完整的 tool calling。

## 2. OpenClaw 架构关键发现

### 2.1 Plugin/Tool 系统

通过 `api.registerTool()` 暴露自定义 tools：

```typescript
// flywheel-bridge plugin
api.registerTool({
  name: "query_sessions",
  description: "Query active Flywheel sessions by status or issue ID",
  parameters: {
    type: "object",
    properties: {
      status: { type: "string", enum: ["running", "awaiting_review", "failed", "blocked"] },
      issue_id: { type: "string" },
    },
  },
  async execute(_id, params) {
    const sessions = store.getActiveSessions(); // 或按 params 过滤
    return { content: [{ type: "text", text: JSON.stringify(sessions) }] };
  }
});
```

可暴露的 tools：
- `query_sessions` — 查询 session 状态
- `get_session_detail` — 获取特定 session 详情
- `approve_pr` — 审批并合并 PR
- `reject_issue` — 拒绝 issue
- `defer_issue` — 延迟处理
- `get_stuck_sessions` — 查询卡住的 sessions
- `ingest_event` — 接收 orchestrator 事件

### 2.2 Sub-Agent 模式

OpenClaw `sessions_spawn` 支持非阻塞子 agent：

```
TeamLead (main agent)
  ├── sessions_spawn("Review GEO-95 PR") → sub-agent #1
  ├── sessions_spawn("Check GEO-96 test failures") → sub-agent #2
  └── Continues main conversation
```

- 子 agent 完成后自动 announce 回主 session
- 层级限制：`maxSpawnDepth: 2`（一层嵌套）
- 并发限制：`maxConcurrent: 8`
- Session key: `agent:<id>:subagent:<uuid>`

### 2.3 多 Agent 配置

```yaml
agents:
  list:
    - id: "product-lead"
      name: "Head of Product"
      workspace: "~/.openclaw/workspace-product"
      model: { primary: "anthropic/claude-sonnet-4-6" }
    - id: "marketing-lead"
      name: "Head of Marketing"
      workspace: "~/.openclaw/workspace-marketing"
      model: { primary: "anthropic/claude-haiku-4-5" }

bindings:
  - agentId: "product-lead"
    match: { channel: "slack", peer: "#product" }
  - agentId: "marketing-lead"
    match: { channel: "slack", peer: "#marketing" }
```

每个 agent 有独立的：SOUL.md、AGENTS.md、memory、tools、session store。

### 2.4 Slack Bridge

| 能力 | 支持情况 |
|------|---------|
| Socket Mode | ✅ 支持 |
| 接收 Block Kit 交互（按钮点击） | ✅ 支持 |
| 发送 Block Kit 富消息 | ❌ 仅 plain text |
| Thread 管理 | ✅ 支持（`:thread:<threadTs>` 后缀） |
| DM | ✅ 支持（多种 scope 策略） |
| Typing indicator | ✅ `ackReaction` + `typingReaction` |
| Access control | ✅ allowlist / pairing / open |

**重要限制**：agent 回复是 plain text，不支持 Block Kit。我们现有的 `TemplateNotifier` Block Kit 通知（带 Approve/Reject 按钮）需要通过 plugin 的 `registerHttpRoute()` 自行发送。

### 2.5 Memory 系统

| | OpenClaw 内置 | mem0 插件 |
|---|---|---|
| 存储 | SQLite + Markdown | pgvector + Neo4j |
| 搜索 | Hybrid（70% vector + 30% BM25） | Semantic |
| 持久化触发 | Model 自行决定 | 自动 capture |
| Recall 触发 | Model 调用 `memory_search` | 自动注入每轮 |
| 抗 compaction | 依赖 pre-compaction flush | 外部存储，天然持久 |
| 部署复杂度 | 零（单文件 SQLite） | 高（3 Docker 容器，4-8 GB） |
| OpenClaw 集成 | 内置 | 官方插件 `@mem0/openclaw-mem0` |

**建议**：Phase 1 用 OpenClaw 内置 memory。如果发现 model-dependent recall 不够可靠（特别是 CIPHER 决策记忆），再加 mem0 插件。

## 3. ⚠️ 关键风险：Subscription vs API Key

### claude-max-api-proxy

| 特性 | 支持 |
|------|------|
| Chat completions | ✅ |
| Streaming | ✅ |
| **Tool use / Function calling** | ❌ **不支持** |
| Models | Opus 4, Sonnet 4, Haiku 4 |

**结论**：proxy 无法支撑 OpenClaw 的 tool use loop — 这是核心功能。

### setup-token 认证

OpenClaw 支持 `claude setup-token`（从已登录的 Claude Code CLI 获取 OAuth token）。**但**：
- Anthropic 2026 年 1 月已开始封杀第三方 harness 使用 subscription token
- Client fingerprinting 已部署
- ToS 已更新，明确禁止非官方客户端
- 首次封禁被撤回，但重复违规可能永久封号

### 直接 API Key

| Plan | 月费 | Token 限制 | Tool Use |
|------|------|-----------|----------|
| Build (Pay-as-you-go) | $0 | 按用量付费 | ✅ 完整 |
| Build + Scale | $100+ | 更高 rate limit | ✅ 完整 |

**建议**：用 API key（Build plan，pay-as-you-go）。TeamLead 场景用量很低（每天几十次 Q&A + 通知），月费可能 < $10。这是最稳定、最完整的路径。

## 4. Options Comparison

### Option A: OpenClaw + API Key (Full)

**Core idea**: 用 OpenClaw 作为 gateway，直接用 Anthropic API key 认证。写一个 `flywheel-bridge` plugin 把 StateStore/ActionExecutor 暴露为 tools。

**Pros**:
- 完整 tool use（read files, exec, memory, sub-agents）
- 持久 session（JSONL），对话记忆
- 多 agent（product-lead, marketing-lead）
- Heartbeat（定时检查 stuck sessions）
- Slack 集成内置（Socket Mode, threads, DM）
- 成熟框架，少造轮子

**Cons**:
- 需要 API key（pay-as-you-go，$5-20/月预估）
- OpenClaw 是外部依赖（版本升级、breaking changes）
- Slack Block Kit 输出不支持 — 需要 plugin workaround
- 学习 OpenClaw 生态（plugin SDK, config, SOUL.md）

**Effort**: Medium（2-3 周）
- Week 1: 安装 OpenClaw + 配置 Slack + SOUL.md
- Week 2: 写 flywheel-bridge plugin（tools + event ingestion）
- Week 3: 多 agent 配置 + Block Kit workaround + 测试

**What gets cut**: 现有 SlackBot, TemplateNotifier, TeamLeadBrain, PromptAssembler, EventIngestion HTTP server

### Option B: OpenClaw + setup-token (Subscription)

**Core idea**: 同 Option A，但用 `claude setup-token` 认证避免 API 费用。

**Pros**:
- 同 Option A 所有优点
- 无额外费用（用现有 subscription）

**Cons**:
- **tool_use 支持未验证** — 可能不工作
- **Anthropic 政策风险** — 可能被封号
- 同 Option A 其他 cons

**Effort**: 同 Option A + 验证 tool_use 兼容性（额外 1-2 天）

**What gets cut**: 同 Option A

### Option C: 自建 OpenClaw-inspired Framework

**Core idea**: 借鉴 OpenClaw 架构（JSONL session, tool loop, memory），用 TypeScript 自己实现。用 `claude -p` 或 SDK。

**Pros**:
- 完全控制，无外部依赖
- 可以用 subscription（`claude -p`）做基础 Q&A
- 可以用 API key 做 tool use（选择性付费）
- 现有 Slack 代码 100% 复用（Block Kit 等）
- 架构完全适配 Flywheel 需求

**Cons**:
- 开发量大（4-6 周）— 本质上在造一个 mini OpenClaw
- JSONL session 管理、context compaction、tool loop 都要自己写
- Sub-agent 协调要自己实现
- Memory 系统要自己实现（或集成现有 v0.3 设计）

**Effort**: Large（4-6 周）

**What gets cut**: 无——但要写大量新代码

### Option D: Hybrid — OpenClaw Gateway + Claude Code Delegate

**Core idea**: OpenClaw 做对话 gateway 和路由，重型 tool 操作（读代码、跑命令）委托给 Claude Code CLI session（subscription）。

**Pros**:
- OpenClaw 处理对话、记忆、routing（用 Haiku，便宜）
- 重型操作用 Claude Code CLI（subscription，免费）
- 结合两者优势

**Cons**:
- 架构复杂（两层 agent）
- Claude Code CLI 交互是异步的（不是 real-time tool call）
- 调试困难

**Effort**: Large（3-4 周）

**What gets cut**: 部分现有代码

---

### Recommendation: Option A (OpenClaw + API Key)

**Rationale**:
1. **Pay-as-you-go API 费用极低** — TeamLead 场景每天几十次调用，月费 < $10
2. **Option B 风险太高** — Anthropic 政策不稳定，可能被封
3. **Option C 开发量太大** — 本质上在重新造 OpenClaw
4. **Option D 太复杂** — 两层 agent 增加调试难度
5. **OpenClaw 生态成熟** — plugin SDK, multi-agent, memory, Slack 都是现成的

### 现有 OpenClaw
- **已有基础配置** — 在现有配置基础上添加 flywheel-bridge plugin 和 product-lead agent。
