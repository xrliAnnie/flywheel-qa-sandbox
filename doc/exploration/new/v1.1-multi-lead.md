# v1.1 Multi-Lead / Multi-Team

> Status: Draft
> Author: Annie + Claude
> Date: 2026-03-10

## Background

Flywheel 目前只有一个 Lead agent（`product-lead`），绑定到 Slack #general（`CD5QZVAP6`）。所有项目的通知和交互都走同一个 channel + 同一个 agent。

Annie 的设想：不同的 Org/Team 有自己的 Lead agent + 专属 Slack channel。每个 issue 根据所属 project 自动路由到对应的 Lead。

## 当前硬编码点

| 位置 | 硬编码内容 |
|------|-----------|
| `event-route.ts` L245-247 | `channel: "CD5QZVAP6"`, `"product-lead"` |
| `hook-payload.ts` buildHookBody | `agentId` 参数 |
| `openclaw.json` hooks.mappings | 单一 mapping → product-lead |
| `run-issue.ts` | `SlackNotifier` channelId 固定 |
| `ApproveHandler.ts` | 通过 projectRoot/projectRepo 匹配，但不区分 lead |

## 设想架构

```
                        ┌─────────────────────┐
                        │   Project Config     │
                        │  (project → lead)    │
                        └─────────┬───────────┘
                                  │
                    ┌─────────────┼─────────────┐
                    ▼             ▼              ▼
            ┌──────────┐  ┌──────────┐   ┌──────────┐
            │product-   │  │eng-      │   │marketing-│
            │lead       │  │lead      │   │lead      │
            │#general   │  │#eng      │   │#marketing│
            └──────────┘  └──────────┘   └──────────┘
```

## 配置模型（草案）

```typescript
interface LeadConfig {
  agentId: string;        // OpenClaw agent ID
  slackChannel: string;   // Slack channel ID
  model?: string;         // 可选覆盖模型
}

interface ProjectEntry {
  projectName: string;
  projectRoot: string;
  projectRepo?: string;
  lead: LeadConfig;       // 新增：对应的 Lead
}
```

Bridge 发事件时从 `ProjectEntry.lead` 读取 `agentId` 和 `slackChannel`，不再硬编码。

## 需要变更的模块

1. **ProjectConfig** — 新增 `lead` 字段
2. **event-route.ts** — 从 project config 读 lead 信息
3. **hook-payload.ts** — `buildHookBody` 接受动态 agentId
4. **openclaw.json** — 多个 hooks.mappings（每个 lead 一个 binding）
5. **run-issue.ts** — `SlackNotifier` channelId 从 project config 读
6. **ApproveHandler** — 确保 approve 通知回到正确的 channel
