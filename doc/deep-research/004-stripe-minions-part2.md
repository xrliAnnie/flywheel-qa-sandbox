# Minions: Stripe's One-Shot, End-to-End Coding Agents — Part 2

## Key Architectural Components

### Devboxes: Infrastructure Foundation

Stripe runs minions on devboxes—standardized AWS EC2 instances containing source code and development services. These environments provide three critical properties:

- **Parallelizability**: Multiple agents can work simultaneously on separate instances
- **Predictability**: Standardized configurations ensure consistent behavior
- **Isolation**: Each agent operates in a quarantined environment, preventing cross-contamination and destructive actions

The infrastructure achieves a "hot and ready" standard where new devboxes spin up within 10 seconds by maintaining a pre-provisioned pool with pre-warmed caches, cloned repositories, and running code generation services. This approach treats devboxes as "cattle, not pets"—replaceable rather than bespoke.

### The Agent Harness

Stripe internally forked Block's Goose coding agent framework in late 2024, customizing it for their LLM infrastructure. The critical distinction from consumer tools like Cursor and Claude Code is the absence of human supervision. As the article explains: "minions are fully unattended, so our agent harness can't use human-facing features such as interruptibility or human-triggered commands."

The quarantined environment enables aggressive optimizations—agents run with full permissions without confirmation prompts since mistakes remain contained.

### Blueprints: Hybrid Orchestration

Rather than pure workflows or pure agent loops, minions use "blueprints"—hybrid orchestration combining deterministic code nodes with agentic decision-making nodes. A typical blueprint includes:

**Deterministic nodes** (rectangles):
- Run configured linters
- Push changes
- Execute code generation

**Agentic nodes** (cloud shapes):
- Implement task
- Fix CI failures

This hybrid approach provides reliability guarantees: "putting LLMs into contained boxes compounds into system-wide reliability upside." Teams can create specialized blueprints for domain-specific tasks, like "LLM-assisted migrations across the codebase that couldn't be accomplished with a straightforward fully deterministic codemod."

### Context Gathering: Rule Files

Minions access contextual guidance through standardized rule files, similar to `.cursorrules` or agent configuration formats. Rather than loading all rules globally (which would exhaust the context window), Stripe implements scoped rules—files automatically attached as agents traverse specific subdirectories or file patterns.

The organization standardized on Cursor's rule format and synchronized it across minions, Cursor, and Claude Code, enabling consistency across all three coding agents.

### Model Context Protocol (MCP) Integration

For dynamic information retrieval, minions use MCP to access:
- Internal documentation
- Ticket details
- Build statuses
- Code intelligence
- External SaaS platforms

Stripe built "Toolshed," a centralized internal MCP server containing nearly 500 tools. Rather than exposing all tools universally, minions receive a "tastefully curated set" relevant to their task, with per-user customization available for additional themed tool groups.

**Security controls** prevent destructive tool usage, though the primary safeguard is that devboxes operate in QA environments without access to production services or real user data.

### Iteration and Feedback Loops

Minions integrate with Stripe's "shift feedback left" philosophy:

1. **Local linting**: Pre-push hooks automatically fix common lint issues via cached heuristics
2. **First CI iteration**: After initial push, minions receive CI results with autofixes applied
3. **Second iteration**: If unfixable failures remain, minions attempt one more local fix
4. **Human handoff**: After second push/CI cycle, branches go to human review

The article notes that "CI runs cost tokens, compute, and time, and we think there are diminishing marginal returns if an LLM is running against indefinitely many rounds of a full CI loop." This policy balances speed against completeness.

Stripe's infrastructure advantage: over three million existing tests provide rich feedback signals, and pre-push hooks eliminate wasted iterations on formatting.

## Scale and Impact

The system produces over 1,300 pull requests weekly (up from 1,000 in Part 1), entirely generated by agents but human-reviewed before merge. No human-written code enters the codebase through minion-produced PRs.

## Philosophy and Lessons

A recurring theme throughout: "What's good for humans is good for agents." Stripe's decades-long investments in developer productivity—devbox infrastructure, caching systems, lint frameworks, test batteries—directly benefited the agent system without redesign.

The article concludes that minions represent "how we're able to blend industry-standard concepts—such as agent harnesses and MCP—with our own mix of internal tooling and infrastructure that our engineers have relentlessly tuned over the years to maximize developer productivity."
